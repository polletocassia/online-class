# Modelo C4

A arquitetura do Sistema de Gestão de Sala de Aula Virtual foi representada utilizando o Modelo C4, que permite visualizar o sistema em diferentes níveis de abstração.

### Contexto (Nível 1)

O diagrama de contexto apresenta a visão geral do sistema, identificando os principais usuários e sistemas externos envolvidos. Nesse nível, os atores Professor e Aluno interagem com o Sistema de Gestão de Sala de Aula Virtual, que por sua vez integra-se aos Serviços de Inteligência Artificial para apoiar a correção automática de atividades e a geração de relatórios.

### Containers (Nível 2)

O diagrama de containers detalha os principais componentes internos da solução:

* Frontend Web: interface utilizada por professores e alunos para acessar as funcionalidades do sistema.  
* Backend/API: responsável pelas regras de negócio, gerenciamento de turmas, atividades, notas e permissões.  
* Banco de Dados: armazena informações de usuários, turmas, atividades, entregas e rankings.  
* Serviços de IA: sistema externo utilizado para correção automática de atividades e geração de relatórios.
